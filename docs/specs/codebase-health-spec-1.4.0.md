## Decisions (locked 2026-07-02 — the former open questions, answered)

1. **--strict blast radius — CONFIRMED as specced, and strict becomes the DEFAULT on the CI/script surface.** The unified contract gates markers, suppressions, flaky/vacuous/skip-in-test-lines, and commented-out blocks; NOT stdout-logging. NEW: on the script/CLI exit-code surface — `check_new_debt.sh` run as a CI step, and the documented CI recipe that gates a `run_audit.sh` step — strict exit codes are now the default, with `--no-strict` (flag) and `WARN_ONLY=1` (env) as the documented escape hatches. *Interpretation note (the split, stated explicitly because it is load-bearing):* strict-by-default applies ONLY to CI/script exit-code behavior. The PostToolUse hook surface (`--hook`) remains warn-only exit 0 UNCONDITIONALLY — hooks-warn-only is a loop-safety invariant (loop-safety.md invariant 3), not a default, and is NOT overturned by this decision; `--hook` ignores strictness flags entirely. See §4.3 for the full contract.
2. **Agent name/model — answered 2026-07-03: no model downgrades, all agents inherit the frontier session model.** Name stands: `test-health-auditor`. NO `model:` frontmatter on the new agent, and Wave 3 REMOVES the existing `model:` pins from all six current agents (architecture-reviewer's `opus`, the five `sonnet` pins) so the whole roster inherits the session model.
3. **/verify rerun budget — N=5 fixed, no override flag.** No `--reruns` option exists anywhere; verify.md documents the fixed budget and its escape-probability rationale only.
4. **Absence-finding severity ceiling — "risk appetite is low, very strict here", read as strict ON THE AUDITED CODEBASE.** *Interpretation note:* "strict" means the audit is strict about dark money paths in the target, not that the plugin caps everything at MED. Both halves encoded crisply in §7: (a) `journey/uninstrumented` may reach HIGH ONLY with a traced CORE money-movement or auth path attached as evidence (the wire-transfer exemplar); (b) ALL other absence findings — module-level logging/correlation-ID absences, DARK/LOG-ONLY vitals off CORE money/auth paths, any absence without the trace — hard-cap at MED needs-verification. No third tier, no judgment escape above the cap.
5. **jscpd — accepted as an optional external tool for `run_audit.sh` on TARGET codebases** (loud `[skip]` degrade, `--min-tokens 50` default). For the self-test, see decision 8: jscpd is a REQUIRED dev dependency there.
6. **Self-test size — accepted.** One script, no slow-lane split. (The draft estimated ~105 assertions; decisions 1/8 and pre-flight fixes F/H moved the recomputed total to ~113 unconditional +3 ruff-conditional — see §"selfTestPlan". Bailey accepted the single-script cost, which only grew.)
7. **Precedence slot — confirmed:** test-health after incomplete-logic, before performance. Chain in §"precedenceChain" unchanged.
8. **ND1 determinism — jscpd becomes a REQUIRED dev dependency for `self_test.sh`.** Hard requirement: the self-test fails loudly at startup if jscpd is absent — no shim-only pass. ND1's near-duplicate pair is therefore scored `expected_by: deterministic` (the pair must appear in `dup_jscpd.json`), with agent corroboration demoted to a lens. `run_audit.sh` keeps the graceful degrade for target repos (we cannot mandate npm on user machines).

**Standing principle from Bailey (applied across the whole spec, not just ND1):** the workflow must be as deterministic and repeatable as possible without constraining agent reasoning. Every `expected_by: agent` fixture and agent-scored claim was swept: wherever a deterministic layer (regex, tool output, fixture assertion) can score it mechanically, primary scoring flipped to deterministic with the agent kept as corroborating lens; where judgment is genuinely required, it stayed agent-scored with the reason recorded. Full sweep results — every flip and every evaluated-but-kept fixture — in §12. This makes the honesty clause stronger, not weaker: the manual-eval-only list shrank and everything remaining on it is there for a stated reason.

Mandate: audit the safety net itself — nondeterminism (a flaky test trains humans to ignore red) and vacuity (a green test that constrains nothing is coverage theater). ### 3.1 references/test-health.md (NEW — merges the flaky taxonomy and the vacuity content the test-quality design had drafted as its own category)

Assertion density is a triage heuristic, never a verdict.
Per-language idiom table (pytest/unittest, jest/vitest, Go testing, Rust #[test]). Framing: coverage % measures execution, not constraint. SEVERITY: HIGH requires demonstrated nondeterminism (bounded probe) OR vacuity-by-construction PLUS sole-coverage of a public-API/security/data-write symbol (traced, or a survived mutant in an ingested report); else MED needs-verification; LOW when real assertions elsewhere cover the path; flaky/vacuous tests are never CRITICAL (no trust boundary) — a rubber stamp blessing a reachable security defect yields a separate, cross-linked security finding. Bounded-probe protocol (single test, N<=10, never destructive). CLOSURE for /verify, strongest first: (nondeterminism) deterministic replacement + bounded stability probe, e.g. 10× green under shuffled order; (vacuity) fresh owner-run mutation report with previously-survived mutants killed, or re-read assertion now substantive + passing run; deletion requires a FIX_LOG/DELETION_LOG entry naming the fingerprint, else STALE; still-unconstraining rewrite → PARTIAL. Guardrails: seeded RNG/frozen clocks/tmp_path/fake timers are FIXES; canned fixture data consumed by real assertions is correct; integration/e2e suites judged by their own contract; bounded polling-with-timeout is not a bare sleep.
