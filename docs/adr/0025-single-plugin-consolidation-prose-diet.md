# ADR 0025 — Consolidate six plugins into one; put the prose on a diet

- **Status:** Accepted (operator approved 2026-07-11)
- **Date:** 2026-07-11
- **Supersedes/amends:** narrows ADR 0011 (marshal as fourth plugin) and ADR 0020 (triage as fifth plugin) — the *capabilities* stand, the *plugin boundaries* do not.

## Context

The suite is feature-complete (v1.2.0) at ~46K lines: six plugins (28.7K), per-plugin
self-tests (~6K), compliance machinery (~4K), and docs (~8K). A structural review against
mattpocock/skills v1.1 (whose entire, widely field-tested workflow is 5.8K lines with a
207-line max file) located the weight in four places, none of them the zero-trust core:

1. **Vendoring generated by the plugin boundary itself.** Independent installability
   forces byte-identical copies: manifest schema ×4, validator toolchain ×3+root,
   escalation-criterion block ×5, claim-overlap ×2, mutation adapter map ×2. Roughly
   half of `lint_consistency.sh` (903 lines, V1–V13) plus planted-drift tests exist
   only to police copies the boundary created.
2. **Prose state machines the model already carries.** 34 reference docs / 5.2K lines
   in autopilot alone; the TDD loop, loop safety, and tracker schema are each explained
   2–3 times in different voices. This is sediment, not specification.
3. **Per-plugin test harnesses** with no shared fixtures (autopilot's alone: 2,327
   lines).
4. **Host adapters with zero shared code** despite a byte-identical contract
   (bitbucket.sh 901 + github.sh 486 lines).

## Decision

1. **One plugin.** Merge {spec-gen, autopilot, codebase-health, marshal, org-memory,
   triage} into a single `zero-trust` plugin: one `skills/`, one `agents/`, one
   `scripts/`, one `schema/` — one copy of everything. Marshal, triage, and org-memory
   become skills/commands + scripts inside it, not plugins. The marketplace registers
   one entry. (Fallback, only if standalone distribution of codebase-health proves
   necessary for adoption: a two-plugin split with health separate — accepted cost:
   one re-vendored schema.)
2. **Retire vendoring enforcement with the vendoring.** Lint rules whose only job is
   byte-identity across plugin copies (V1, V3, V4, V5, V7, V8) are deleted alongside
   the copies. Structural/format rules (behavior-coverage format, marketplace
   registration, telemetry contract, etc.) survive.
3. **Prose diet under an explicit skill-writing discipline.** Adopt the
   writing-great-skills vocabulary (no-op, sediment, sprawl, leading word, progressive
   disclosure, negation) as this repo's editing standard for skills and references.
   Skills state only what deviates from the model's defaults; lifecycles are anchored
   by leading words, not restated; reference goes behind pointers.
4. **Invocation taxonomy.** Every skill declares itself user-invoked (orchestrator,
   `disable-model-invocation: true`, zero context load) or model-invoked (reusable
   discipline). A user-invoked skill may invoke model-invoked skills, never another
   user-invoked one.

## What this ADR explicitly protects (non-goals)

- **The deterministic substrate is untouched in semantics:** manifest schema v1 and
  the 8-rule completeness checker, refuse-to-finalize, propose-confirm, the D6.5
  mutation gate, N=5 closing-test determinism, marshal's zero-judgment FIFO,
  outcome honesty-class badges. The thesis is *keep the gates, gut the prose*.
- **No tracker-as-state rework.** The wayfinder pattern (plan-as-tracker-issue with
  typed blocking sub-issues) is attractive but the production target is Bitbucket
  Data Center, not GitHub Issues. Autopilot's file-based runbook/tracker substrate
  stays. Revisit only with a Jira-DC or local-markdown backend in hand.
- **No manifest schema version bump.**

## Consequences

- Suite version bumps to v2.0.0 (breaking: plugin names/install paths). README carries
  a reinstall migration note.
- `suite_self_test.sh` shrinks (one harness, shared fixtures) but the assertion count
  is a floor, not a casualty: no coverage loss is accepted in any wave.
- Execution is staged in waves, each an isolated PR through the established
  chip-build → adversarial-review → SUITE_STRICT-green gate; see
  `docs/specs/refactor-2026-07-consolidation.md`.
- Risk: the prose diet can delete load-bearing steering. Mitigation: waves that touch
  skill semantics get a definition-review pass and a field run before the next wave
  starts.
