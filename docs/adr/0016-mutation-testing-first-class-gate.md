# ADR 0016 — Mutation testing is a first-class changed-files-scoped-then-changed-lines-filtered gate at D6.5 and the PR Gate, run only in an isolated throwaway worktree drain-side and ingest-only audit-side

---
status: agent-decided
date: 2026-07-06
---

Mutation testing is the technique the suite's own adversarial reviews already lean on: a *survived mutant is deterministic proof a test constrains nothing* (SPEC_1.4.0 §7 / cross-language-tooling.md — the gold-standard vacuity evidence). Today the suite only INGESTS owner-run mutation reports passively (run_audit.sh copies mutmut/Stryker/cargo-mutants/go-mutesting output into `audit/` as `mutation_stryker.json` / `mutation_cargo_missed.txt` / `mutation_go.txt` / `mutation_mutmut.txt`; absent → loud `[note]`; the codebase-health SPEC invariant 1 forbids the AUDIT from ever RUNNING a mutation tool because it mutates the working tree). Passive ingestion answers *did someone run it* — it never makes a survivor cost anything at the moment a vacuous test is written or a weak test guards a money path. This ADR elevates the technique to a first-class GATE at the two points where the suite already enforces test quality, WITHOUT overturning the audit's ingest-only invariant, and WITHOUT inventing parallel infrastructure.

**Two active enforcement points.**

1. **Autopilot D6.5 — anti-vacuous gate (complement to D6.4 anti-flaky).** After a Subtask's tests are green (D6.1) and proven deterministic across N=5 rounds (D6.4), a NEW D6.5 runs a mutation tool over the Subtask's OWN changed product FILES, then filters the survivors to the changed LINES (`prev_pushed_sha..HEAD`, the same range D6.2 audits). A survivor on a changed line means a test that executes that line asserts nothing about it — coverage theater. Verdict: `[BLOCKED: vacuous-test]` (impl-block counter, `consecutive_impl_blocks`, re-arm `*/30`, escalate at `max_impl_blocks=3` — dispatch identical to D6.4's `[BLOCKED: flaky-test]`). D6.4 catches a test that passes for the wrong reason (nondeterminism); D6.5 catches a test that passes for NO reason (vacuity).

   **Isolation is a NEW named mechanism, not free reuse.** A mutation tool rewrites source on disk, so D6.5 must NOT run on the live Story checkout (that would violate autopilot loop-safety invariant 1 — probes never mutate operator-visible state — and could leave a half-mutated tree on crash). D6.5 runs inside an EXPLICIT `git worktree add <throwaway> HEAD` with an EXIT/INT/TERM trap that force-removes it, modeled on repo_shape_probe.sh's trap discipline and gated behind a clean-index precheck. The live checkout is never touched. This is the honest cost of first-classing the technique: a small isolation harness, not zero infra.

2. **The PR Gate — a NEW survivor on a CORE money/auth changed line blocks (ADR 0004 deterministic-evidence class).** The PR Gate gains a diff-scoped mutation sibling composed into the real `pr_gate.sh` `run_sibling`+`[ -x ]`+`[not-covered]` pattern (alongside check_new_debt.sh / check_memory_rot.sh). It READS the INGESTED report (never runs the tool — this side IS the audit, invariant 1) and joins survived mutants against the diff's changed lines AND the manifest/journeys criticality. A NEW survivor on a changed line the journey trace grades CORE money/auth is the ADR-0004 blocking class; a survivor anywhere else (SUPPORTING/DEV, untraced, or off-diff) is comment-only. Ratchet per ADR 0004: only NEW survivors on the diff's changed lines block; inherited survivors never do.

**Honesty about determinism (de-laundered).** The block is deterministic *in the join* — survivor is a fact in an ingested artifact, the changed line is `git diff`, the criticality is a *recorded field value* in a prior journeys.json — exactly how CH-05 memory-rot and CH-06 coverage gate on prior-artifact field values. But that criticality VALUE was AGENT-DERIVED upstream (SPEC 'criticality (derived)'; CH-03 marks derivation `[audit-run]`, agent-scored). Therefore, per ADR 0004's own invariant *an agent opinion without deterministic evidence never blocks a human's merge*, this class CAPS AT COMMENT-ONLY whenever journeys.json is absent, degraded, or its criticality is `unknown`. The gate never guesses criticality (SPEC degrade rule).

**The SLOW constraint is a hard scoping law — and the tool support is uneven, stated honestly.** Whole-repo mutation is quadratic and unusable as a gate. Scoping:
- **Changed FILES only, then filter to changed LINES — never whole-repo.** Only cargo-mutants (`--in-diff`) and StrykerJS (`--incremental`/`--mutate` ranges) support line/diff scoping at the tool level; mutmut (`--paths-to-mutate`, FILE granularity, no diff mode) and go-mutesting (no incremental mode) scope to changed FILES, and the survivor→changed-LINE filter is applied post-hoc in the join. Claiming line-level tool scoping for all four would be false.
- **A mutant/time budget** is the real affordability backstop for the two file-granular tools: D6.5 honors `budget.max_mutants_per_subtask` (default 40) and `budget.max_mutation_seconds` (default 120); exceeding either is `[note] mutation-budget-exhausted — partial (N of M)`, exit 0, NEVER a false `[BLOCKED]` (inconclusive ≠ survivor — D6.4's skipped-randomization honesty). These defaults are AGENT-DECIDED (reversible runbook edits, downstream-verifiable — a wrong value degrades to a `[note]`, never a false block), overturnable async; they are NOT escalated.
- **Critical paths only for the blocking verdict.** Even within the diff, only survivors on traced CORE money/auth lines gate; everything else is comment-only.

**Self-remediation is closed.** A `[BLOCKED: vacuous-test]` re-spawns a fix, bounded by `max_impl_blocks`. The remedy MUST be a strengthened assertion re-verified by D6.5 on the SAME changed lines — NOT deleting the product code at the mutation point (which trips the existing D6.2 tdd-scope-leak audit) and NOT editing `gates.test_mutation` to dodge the gate (out of the implementer's `owned_files[]` window). D6.5 adds no new autonomous mutating path beyond the trap-isolated throwaway worktree.

**Per-language tools + survivor location + graceful degrade (loud [note], never silent).** The adapter map is the cross-language-tooling.md table (mutmut/cosmic-ray · StrykerJS · cargo-mutants · go-mutesting) plus each tool's changed-FILES invocation AND its survivor→location resolver (mutmut needs `mutmut show`, not just `mutmut results`, to get file:line; where a tool yields no line, the survivor degrades to file granularity and caps at comment-only on the PR side). No tool for the language, or `gates.test_mutation` omitted → D6.5 SKIPS with a loud `[note] no mutation tool for <lang> — D6.5 anti-vacuous gate skipped (optional)` on stderr, exit 0 — exactly like D6.4 skipping the order-randomized round when `gates.test_random` is unset. The PR-Gate side degrades like the existing ingestion: no report → loud `[note]`, mutation facet adds itself to Not-covered, never blocks.

**Reconciliation.** The audit's ingest-only path is unchanged and stays authoritative for the WHOLE-REPO picture (ambient audit + test-health-auditor's survived-mutant triage). The two gate points are the CHANGED-LINES complement: D6.5 PRODUCES a fresh per-Subtask signal drain-side (the ONLY place a tool RUNS, trap-isolated); the PR-Gate sibling CONSUMES the ingested report and scopes it to the diff. One tool family, one adapter map (pinned by lint), three consumers — no parallel infrastructure, no second wrapper.

## Considered Options

- **Leave it ingest-only (status quo).** Rejected: a survivor becomes a finding only if a human ran the tool out-of-band and committed the report — the technique the suite's reviews rely on is not enforced by the suite. First-classing gives vacuity a cost at write time (D6.5) and merge time (PR Gate).
- **Run mutation whole-repo in the audit.** Rejected twice: violates invariant 1 (audit must never mutate the tree) AND is quadratically slow. Whole-repo stays owner-run + ingest-only.
- **Run D6.5 on the live Story checkout.** Rejected: violates autopilot loop-safety invariant 1 and risks a half-mutated tree on crash. The throwaway-worktree-with-trap is the isolation cost that makes a drain-side tool run safe.
- **Claim line-level tool scoping for all four languages.** Rejected as false: mutmut and go-mutesting are file-granular. The honest design scopes to files and filters survivors to lines, with the budget cap as the affordability backstop.
- **A standalone mutation-testing plugin.** Rejected for the ADR-0003 reason a standalone PR-review plugin was rejected: it would duplicate the adapter map + criticality join behind a second implementation that drifts. D6.5 reuses determinism_gate.sh + repo_shape_probe.sh trap discipline; the PR-Gate sibling reuses pr_gate.sh composition. No fourth checker.
- **Block on a repo-wide mutation-SCORE threshold.** Rejected as an agent decision — a numeric block-or-pass threshold is a values/risk-appetite call (ADR 0002) AND a repo-wide score contradicts the changed-lines scoping law and ADR 0004's new-debt-only ratchet. Flagged for Bailey (⟨MT-AMEND-B⟩), not decided here.

## Consequences

- Autopilot D6 grows D6.5 and `scripts/mutation_gate.sh` (runner-agnostic, modeled on determinism_gate.sh, isolated via a trap-torn-down throwaway `git worktree`); the runbook `gates:` block gains `test_mutation` (OPTIONAL) and `budget:` gains `max_mutants_per_subtask`(40) + `max_mutation_seconds`(120). Absent config → skip + loud `[note]`, never a false block.
- Codebase-health's `pr_gate.sh` gains a `check_mutation_survivors.sh` sibling (its OWN strict-default + `--no-strict`/`WARN_ONLY=1` contract, mirroring check_new_debt.sh — a sibling owns its exit code; pr_gate.sh stays warn-only) that joins the ingested report to the diff's changed lines and the traced criticality; a CORE money/auth survivor is the ADR-0004 blocking class, capped at comment-only when the trace is absent/degraded.
- The root `lint_consistency.sh` gains V7 pinning the adapter map via a delimited vendored block (`<!-- vendored:mutation-adapter-map:begin/end -->`, the V5 mechanism — NOT a semantic 'referenced identically' assertion the lint cannot enforce) and grep-pinning the `[BLOCKED: vacuous-test]` producer token and the `mutant-on-core-path` consumer token so producer and consumer cannot drift.
- **Flagged for Bailey (not applied):** (a) ⟨MT-AMEND-A⟩ whether the CORE-survivor class is promoted into ADR 0004's SHIPPED blocking defaults vs report-only (ships report-only during the ADR-0004 soak either way, per the ⟨CH-AMEND-C⟩ precedent); (b) ⟨MT-AMEND-B⟩ whether any repo-wide mutation-SCORE threshold ever gates. The D6.5 budget defaults are NOT flagged — they are agent-decided (reversible + downstream-verifiable per ADR 0002).
